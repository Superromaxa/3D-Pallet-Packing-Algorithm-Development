# 3D Pallet Packing Algorithm Development

Набор C++ утилит для укладки коробок на паллету, анализа результата и восстановления исходного набора коробок из файла размещения.

## Основные файлы

| Файл | Назначение |
| --- | --- |
| `main_candidate_heights.cpp` | Слойная укладка: выбирает высоты слоев-кандидатов и размещает коробки внутри слоя через MaxRects. |
| `stable_pallet_support60.cpp` | Укладка с проверкой устойчивости: центр коробки должен иметь опору, площадь опоры не меньше 60%. |
| `stable_pallet_support70.cpp` | Жадная устойчивая укладка: проверяет локальную опору коробки и старается держать поддержку не ниже 70%. |
| `stable_pallet_ga70.cpp` | Генетическая версия: оптимизирует порядок коробок, а декодер ставит их с целевой поддержкой 70%. |
| `pallet_analyzer.cpp` | Анализ готовой укладки: заполнение объема, пересечения, выход за границы паллеты, поддержка коробок, центр масс. |
| `unpack.cpp` | Обратное восстановление списка коробок из CSV с координатами размещения. |

## Сборка

```bash
cmake -S . -B build
cmake --build build
```

После сборки появятся четыре исполняемых файла:

```text
build/candidate_heights
build/stable_pallet_support60
build/stable_pallet_support70
build/stable_pallet_ga70
build/stable_multi_pallet_ga70
build/pallet_analyzer
build/unpack
```

## Запуск

### Слойная укладка

```bash
./build/candidate_heights
```

Программа ожидает входные файлы `order_2.csv` ... `order_16.csv` в текущей рабочей директории и пишет результаты вида `order_K_outc++_candH.csv`.

### Устойчивая укладка

```bash
./build/stable_pallet_support60
```

Программа ожидает набор заказов `ORDERS436/*.csv` и пишет результаты вида `order_K_out.csv`.

### Устойчивая укладка с целевой поддержкой 70%

```bash
./build/stable_pallet_support70 "Orders 0802 2011" 153 153 order_
```

Жадный вариант: быстрее, но может терять качество на сложных заказах.

### Генетическая укладка с целевой поддержкой 70%

```bash
./build/stable_pallet_ga70 "Orders 0802 2011" 153 153 order_ 24 20
```

Аргументы: папка заказов, первый номер, последний номер, префикс результата, размер популяции, число поколений. Для небольших заказов около 100 коробок разумный старт: `24 20`.

### Генетическая укладка на несколько паллет

```bash
./build/stable_multi_pallet_ga70 "Orders 0802 2011" 153 153 order_ 24 20
```

Число паллет читается из первой непустой строки входного файла. Для быстрого эксперимента можно передать override последним аргументом:

```bash
./build/stable_multi_pallet_ga70 "Orders 0802 2011" 153 153 order_ 24 20 2
```

Результат содержит первую строку с числом паллет, общую перколяцию по формуле `total_volume / (1200 * 800 * sum_pallet_heights)` и далее CSV-таблицу `Pallet,SKU,x1,y1,z1,x2,y2,z2,weight,aisle,support`.

### Анализ укладки

```bash
./build/pallet_analyzer placements.csv 1200 800
```

Формат входного CSV:

```text
SKU,x1,y1,z1,x2,y2,z2[,weight][,aisle]
```

### Восстановление коробок из укладки

```bash
./build/unpack placement.csv reconstructed_boxes.csv --merge
```

Флаг `--merge` объединяет одинаковые коробки в одну строку с количеством.

## Текущий статус

Репозиторий пока хранит четыре самостоятельные CLI-утилиты. Следующий хороший шаг по коду - вынести общий парсинг CSV, модель коробки и формат вывода в общие модули, чтобы алгоритмы укладки, анализатор и распаковка использовали один контракт данных.
