# 3D Pallet Packing Algorithm Development

Набор C++ утилит для укладки коробок на паллету, анализа результата и восстановления исходного набора коробок из файла размещения.

## Основные файлы

| Файл | Назначение |
| --- | --- |
| `main_candidate_heights.cpp` | Слойная укладка: выбирает высоты слоев-кандидатов и размещает коробки внутри слоя через MaxRects. |
| `stable_pallet_support60.cpp` | Укладка с проверкой устойчивости: центр коробки должен иметь опору, площадь опоры не меньше 60%. |
| `pallet_analyzer.cpp` | Анализ готовой укладки: заполнение объема, пересечения, выход за границы паллеты, поддержка коробок, центр масс. |
| `unpack.cpp` | Обратное восстановление списка коробок из CSV с координатами размещения. |

## Сборка

```bash
cmake -S . -B build
cmake --build build
```

После сборки появятся четыре исполняемых файла:

```text
build/candidate_heights
build/stable_pallet_support60
build/pallet_analyzer
build/unpack
```

## Запуск

### Слойная укладка

```bash
./build/candidate_heights
```

Программа ожидает входные файлы `order_2.csv` ... `order_16.csv` в текущей рабочей директории и пишет результаты вида `order_K_outc++_candH.csv`.

### Устойчивая укладка

```bash
./build/stable_pallet_support60
```

Программа ожидает набор заказов `ORDERS436/*.csv` и пишет результаты вида `order_K_out.csv`.

### Анализ укладки

```bash
./build/pallet_analyzer placements.csv 1200 800
```

Формат входного CSV:

```text
SKU,x1,y1,z1,x2,y2,z2[,weight][,aisle]
```

### Восстановление коробок из укладки

```bash
./build/unpack placement.csv reconstructed_boxes.csv --merge
```

Флаг `--merge` объединяет одинаковые коробки в одну строку с количеством.

## Текущий статус

Репозиторий пока хранит четыре самостоятельные CLI-утилиты. Следующий хороший шаг по коду - вынести общий парсинг CSV, модель коробки и формат вывода в общие модули, чтобы алгоритмы укладки, анализатор и распаковка использовали один контракт данных.
